# 竞赛备战规划 Agent — 产品需求文档（PRD）v0.1

> 适用：校招 AI 产品实习作品集 · 首版只支持 **1 个比赛模板**（见文末「模板 A」示例）  
> 目标用户：准备参加某类竞赛的大学生（入门～有基础）  
> 核心价值：输入比赛信息 → 输出可执行的备战包（路径 + 分级资料 + 论文 + Checklist）

---

## 1. 背景与问题

### 1.1 用户痛点
- 官方信息、教程、论文、往届题分散在多个平台，不知道**先读什么、读多少**。
- 临近截止缺少**按周倒排**的里程碑。
- 通用 ChatGPT 容易**编造链接**、推荐与赛题无关的论文。

### 1.2 产品定位（一句话）
**竞赛备战规划 Agent**：解析赛题与能力要求，生成分阶段学习路径和分级参考资料（含论文），支持导出 Markdown 备战包。

### 1.3 非目标（v1 不做）
- 不做每日陪练、自动出题、自适应难度。
- 不做全站爬虫；不做「支持所有比赛」。
- 不承诺上线级内容审核或官方合作。

### 1.4 成功指标（MVP）
| 指标 | 目标（试用 N≥10 人后填写） |
|------|---------------------------|
| 主观有用率（1–5 分 ≥4） | ≥ 60% |
| 用户愿意导出/转发备战包 | ≥ 50% |
| 胡编链接率（抽检 20 条输出） | ≤ 5% |
| 生成耗时（P95） | ≤ 90 秒 |

---

## 2. 用户与场景

### 2.1 用户画像
- 本科生 / 研究生，计划 4–8 周内参加比赛。
- 每周可投入 5–20 小时（用户自填）。
- 当前水平：入门 / 有基础 / 有参赛经验（三选一）。

### 2.2 核心用户故事
1. 作为备赛学生，我输入**比赛名称 + 官方链接 + 截止日期**，希望得到**分阶段路径**，以便按周执行。
2. 作为备赛学生，我希望看到**必读官方资料 + 推荐教程 + 3–5 篇相关论文**，并知道**为什么推荐**。
3. 作为备赛学生，我希望一键**导出 Markdown**，方便打印或发给队友。
4. 作为备赛学生，我标记「已读」后，下次打开能看到**进度**（v1.1）。

## 3. 范围：功能清单

### 3.1 Must Have（v1，2–3 周）
| ID | 功能 | 说明 |
|----|------|------|
| F1 | 比赛信息 Intake | 表单：比赛名、官方 URL、赛题文本（可选粘贴）、截止日期、每周小时、水平 |
| F2 | 赛题解析 | 结构化输出：赛制、评分、提交物、能力项、常见坑 |
| F3 | 分阶段路径 | 4–6 阶段，每阶段含目标、技能点、时长、验收标准 |
| F4 | 分级资料推荐 | 官方 / 教程&项目 / 论文 三层，每条含链接、理由、时长 |
| F5 | 论文检索增强 | arXiv 或 Semantic Scholar 关键词检索 + 与 curated 合并 |
| F6 | 能力差距清单 | 赛题要求 vs 用户自评 → 缺什么 |
| F7 | 提交物 Checklist | 按比赛类型生成（论文赛/代码赛/答辩赛） |
| F8 | 导出 Markdown | 一键下载 `.md` |
| F9 | 防幻觉规则 | 无 verified 链接则标 `待补充`，禁止编造 URL |

### 3.2 Should Have（v1.1，+1 周）
| ID | 功能 |
|----|------|
| F10 | 保存历史方案 + 已读/收藏 |
| F11 | 推荐质量反馈（1–5 分 + 无用条目 ID） |
| F12 | 第二个比赛模板（复制 workflow） |

### 3.3 Won't Have（v2 以后）
- 每日推送、社区、组队匹配、自动爬取知乎全文。

## 4. 用户流程

```
注册/匿名 → 填写 Intake → [可选] 拉取官方页摘要
    → Agent 流水线（解析 → 路径 → 资料 → 论文 → 合并）
    → 结果页展示 → 用户勾选已读 / 评分
    → 导出 Markdown → 结束
```

**异常流**
- 未提供官方链接：仍可根据粘贴的赛题文本生成，但资料层「官方」标为待补充。
- API 超时：展示部分结果 +「重试」按钮。
- 论文 API 失败：仅展示 curated 论文，并提示。

## 6. Intake 输入字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `competition_name` | string | 是 | 如「2026 校际 AI 创新大赛」 |
| `competition_template_id` | enum | 是 | `ai_innovation` / `kaggle_tabular` / `math_modeling`（v1 只开 1 个） |
| `official_url` | url | 否* | *强烈建议填；用于摘要与可信度 |
| `rules_text` | text | 否 | 粘贴赛题/章程原文，与 URL 至少其一 |
| `track` | string | 否 | 赛道，如「NLP」「视觉」 |
| `deadline` | date | 是 | 截止日期 |
| `weekly_hours` | int | 是 | 5–40 |
| `skill_level` | enum | 是 | `beginner` / `intermediate` / `experienced` |
| `team_size` | int | 否 | 默认 1 |
| `self_assessed_skills` | string[] | 否 | 如 `["Python","PyTorch"]` |
| `goal` | string | 否 | 如「进省赛」「完成 baseline」 |

## 7. 输出数据结构

完整 JSON Schema 见同目录 `competition-prep-output-schema.json`。

**流水线顺序**
1. `competition_analysis`（Prompt 1）
2. `prep_plan`（Prompt 2，依赖 1）
3. `resource_recommendations`（Prompt 3 + curated DB + 论文 API）
4. `merge` → 最终 `prep_package`

## 8. 资料策略（PM 必做）

### 8.1 可信度分层
| 层级 | 标签 | 来源示例 |
|------|------|----------|
| L0 | `official` | 官网、组委会 PDF、赛题页 |
| L1 | `course` | 官方教程、知名课程、官方 GitHub |
| L2 | `community` | 博客、知乎专栏（仅作补充，标注非官方） |
| L3 | `paper` | arXiv、Semantic Scholar |

### 8.2 Curated 种子库（v1 最低要求）
- 为你选的 **1 个比赛** 准备 **30–50 条** 链接（CSV：`title, url, tier, tags, notes`）。
- 论文 **5–10 篇** 人工标「必读/选读」。

### 8.3 论文检索
- 从 `competition_analysis.required_skills` + `track` 生成 3–5 个英文关键词。
- 调用 arXiv API，取近 3 年、最多 10 条，再由 LLM 筛 3–5 条写 `relevance_reason`。
- **禁止** LLM 单独编造 DOI/arXiv ID；无 API 结果则只输出 curated。

## 9. Agent 工作流（技术建议）

```
[Intake] → [Fetch 官方页纯文本]（可选，trafilatura/简单 requests）
    → [LLM Prompt1: 赛题解析] → JSON
    → [LLM Prompt2: 路径规划] → JSON
    → [检索 curated CSV + RAG 可选]
    → [arXiv API]
    → [LLM Prompt3: 合并排序+写理由] → JSON
    → [渲染 UI + 导出 MD]
```

**推荐栈（vibe coding）**
- 前端：Vue 3 + Element Plus（FastAPI 后端）
- 编排：LangGraph / Dify / 纯 Python 顺序调用
- 存储：SQLite（`plans` 表存 JSON）
- 模型：DeepSeek / 通义 / 豆包 API 任选

## 10. 评测与迭代

### 10.1 人工评测集（20 条）
每条：Intake 输入 + 你手工写的「理想路径要点」+ 必含资料类型。  
生成结果按 1–5 分打：**赛题理解 / 路径可执行 / 资料相关性 / 无胡编链接**。

### 10.2 Bad Case 分类
- `hallucinated_link`：链接不存在
- `off_topic`：与赛道无关
- `too_hard`：路径超出截止时间
- `too_vague`：验收标准不可执行

### 10.3 版本记录
`prompt_version` 字段写入每次输出，便于简历写「v1→v2 有用率提升 X%」。

---

## 11. 合规与风险

- 不爬取需登录或明确禁止抓取的站点；优先用户粘贴 + 官方公开页。
- 论文仅输出公开链接；摘要可引用，不全文搬运版权材料。
- 结果页免责声明：「仅供参考，以官方章程为准」。

## 13. 附录：模板 A — AI 创新/应用类校赛（首版默认）

**`competition_template_id`**: `ai_innovation`

**典型能力项**（供 Prompt 参考，非固定）  
- 问题定义与方案设计  
- 数据处理与特征工程  
- 模型训练与评估

- [ ] 训练数据说明与来源合规  
- [ ] 对比实验 / 指标表  
- [ ] 答辩 PPT（5–8 分钟）  

**curated 标签建议**  
`python`, `ml_basics`, `nlp`, `cv`, `deployment`, `ppt`, `ethics`

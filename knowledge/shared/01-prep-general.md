# 竞赛备赛通用指南（84 项适用）

本条目供 RAG 检索，为各类学科竞赛提供可复用的备赛框架。具体赛制以主办方最新通知为准。

## 阶段一：读懂赛制与提交物

- 确认赛段：校赛 → 省赛/区域赛 → 国赛/决赛，各自截止时间与晋级规则。
- 明确提交物：论文、代码、作品、视频、答辩 PPT、商业计划书等，以及格式与大小限制。
- 记录评分维度：创新性、完整性、技术难度、展示表达、商业可行性等权重。
- 若有子赛道/组别，选定与团队能力最匹配的一条，避免中途换题。

## 阶段二：补齐基础技能

- 对照赛项要求列出技能清单（工具、语言、理论、硬件等）。
- 按「缺口最大 + 赛题必用」排序，每周固定投入小时数（参考用户 Intake 中的 `weekly_hours`）。
- 优先完成 1～2 个最小可行练习（Mini Project），而非只看教程。
- 团队分工：明确谁负责建模/开发/文档/答辩，并约定沟通节奏。

## 阶段三：完成练习与模拟

- 使用往年真题、公开数据集或模拟题进行限时演练。
- 模拟应覆盖完整流程：读题 → 方案 → 实现/撰写 → 打包提交。
- 记录耗时与卡点，赛前至少完成 1 次全真模拟。
- 校赛/练习赛是检验流程与分工的最佳机会，不必等到省赛才组队磨合。

## 阶段四：打磨作品与文档

- 对照官方评分细则逐项自检，补全摘要、说明书、演示视频等易漏项。
- 图表、目录、引用格式统一；代码仓库 README 说明运行环境与复现步骤。
- 预留 24～48 小时缓冲，用于格式转换、上传失败重试、材料补交。

## 阶段五：预答辩或模拟答辩

- 准备 5～10 分钟陈述 + Q&A，覆盖：问题定义、方案、亮点、局限与改进。
- 邀请同学或导师扮演评委，收集「听不懂的点」并改表述。
- 答辩材料与提交物版本一致，避免现场展示与提交版本不符。

## 阶段六：正式提交与复盘

- 提交前核对：文件名、账号、截止时间（含时区）、是否需要邮寄或现场注册。
- 保留提交截图或回执邮件。
- 赛后复盘：时间分配、技术选型、团队协作，记录供下一届或队友参考。

## 与时间预算相关的建议

| 距离截止 | 建议重点 |
|----------|----------|
| ≥ 8 周 | 系统学习 + 2 次模拟 |
| 4～8 周 | 以赛题/真题驱动，减少泛学 |
| ≤ 4 周 | 聚焦提交物清单与 1 次完整模拟，不追求新技术栈 |
| ≤ 1 周 | 整理已有成果、补文档与答辩，避免大规模重构 |

## 与 Agent 生成的关系

- 用户选定 catalog 中的比赛后，Agent 会结合 `archetype` 赛道模板与本通用阶段生成个性化路径。
- 若用户提供了 `rules_text` 或官网摘要，以其为准覆盖本指南中的泛化描述。
- 无 verified 来源的链接不得编造；不确定时标注「待补充」。
